# 拼豆图转 Excel 像素画

> 一款纯前端 Web 工具，可将任意图片转换为拼豆（Pixel Bead）设计图，并导出为 Excel 文件，方便按图拼豆。

## 快速开始

**双击 `index.html` 即可在浏览器中直接使用**，无需安装任何依赖或启动服务器。

> 所有第三方库（Cropper.js、ExcelJS）已内置在 `lib/` 目录，完全离线可用。

如需通过 HTTP 服务器访问（可选）：

```bash
# Python
python -m http.server 8000

# Node.js
npx serve .
```

## 功能特性

### 核心功能

- **图片上传** — 支持文件选择器和**拖拽上传**两种方式
- **格子校准** — 框选一颗豆子即可自动算出整张图的行列数，无需手动输入参数
- **实时网格** — 裁切框上叠加实际的拼豆网格线，所见即所得
- **比例锁定** — 支持锁定/解锁裁切框比例，适应"已知行列"和"视觉对齐"两种场景
- **LAB 色彩匹配** — 使用 CIE LAB 色彩空间 + Delta E 色差算法，将图片颜色精确映射到拼豆色卡
- **多色卡支持** — 内置 Artkal S（159 色）和 MARD（291 色）两套官方色卡
- **Excel 导出** — 生成 `.xlsx` 文件，每个单元格对应一颗拼豆，含色号、填色、边框、图例

### 交互体验

- **实时预览** — Canvas 渲染拼豆效果，支持 50%~500% 缩放（按钮 / 滚轮）
- **悬浮提示** — 鼠标悬停预览区域显示当前格子的色号、RGB 值
- **色豆统计** — 自动统计每种颜色用量，降序排列
- **操作引导** — 首次访问显示 3 步引导栏（可关闭，记忆到 localStorage）
- **移动端适配** — 编辑/浏览模式切换，响应式布局
- **快捷键** — `Enter` 生成预览，`Ctrl+E` 导出 Excel

### 智能参数

| 参数 | 范围 | 默认值 | 说明 |
|------|------|--------|------|
| 列数 | 1~200 | 26 | 拼豆画水平方向的格子数 |
| 行数 | 1~200 | 23 | 拼豆画垂直方向的格子数 |
| 像素/格 | 2~500 px | 自动 | 每个拼豆格对应原图的像素数，通过 +/- 按钮或校准自动设定 |
| 比例锁定 | 🔒/🔓 | 锁定 | 锁定：列/行控制裁切框比例；解锁：自由拖拽裁切框 |
| 单元格大小 | 8~48 px | 24 | Excel 中每个格子的像素尺寸 |
| 钉板尺寸 | 1~100 格 | 28 | 每块物理钉板的边长，用于绘制粗线辅助线 |
| 色卡 | — | MARD | 使用的拼豆品牌色卡 |
| 忽略白色 | 开/关 | 关 | 是否将接近白色的像素标记为空白 |
| 灵敏度 | 70~98 | 90 | 白色检测阈值（仅勾选"忽略白色"后可见） |
| 显示色号 | 开/关 | 关 | 预览画布上是否叠加显示色号文字 |
| 仅填色 | 开/关 | 关 | 导出时是否省略色号文字，只保留单元格填色 |
| 图例 | 开/关 | 开 | 导出时是否在右侧附带色号/数量/颜色图例 |

## 使用流程

### 方式一：格子校准（推荐）

无需知道图片尺寸或行列数，纯视觉操作：

```
① 上传图片 → ② 框选一颗豆子 → ③ 点"设为一格" → ④ 框选识别范围 → ⑤ 生成预览 → ⑥ 导出 Excel
```

1. **上传图片** — 点击文件选择器或将图片拖拽到左侧区域
2. **框选一颗豆子** — 拖动裁切框恰好框住图中的一个拼豆格子
3. **设为一格** — 点击左侧面板的「📐 设为一格」按钮，裁切框自动放大到整张图
4. **框选识别范围** — 拖动裁切框选择要转换的区域，行列数自动实时计算
5. **生成预览** — 点击"生成预览"或按 `Enter`，查看拼豆效果
6. **导出 Excel** — 满意后点击"导出 Excel"或按 `Ctrl+E` 下载文件

### 方式二：手动参数

适合已知行列数或需要精确控制的场景：

```
① 上传图片 → ② 设置列/行 → ③ 框选区域 → ④ 生成预览 → ⑤ 导出 Excel
```

1. **上传图片** — 同上
2. **设置参数** — 输入列数、行数，或通过像素/格的 +/- 按钮调整网格密度
3. **框选区域** — 裁切框自动锁定为列/行比例，拖动选择区域
4. **生成预览** — 同上
5. **导出 Excel** — 同上

> 修改参数后导出时，系统会自动提醒重新生成预览，避免导出过期数据。

## 色卡数据格式

色卡文件导出一个常量数组，每个元素包含：

```javascript
const PALETTE_ARTKAL_S = [
  { code: 'S01', rgb: [255, 255, 255] },
  { code: 'S02', rgb: [245, 245, 220] },
  // ...
];
```

- `code` — 色号（品牌官方编号）
- `rgb` — RGB 三通道值 `[R, G, B]`（0~255）

如需添加新色卡，创建新文件并在 `app.js` 的 `PALETTES` 对象中注册即可。

## 核心算法

### LAB 色彩匹配

1. 将图片裁切区域划分为 `列 × 行` 个网格
2. 对每个网格的中心 65% 区域取像素 RGB 均值
3. 将 RGB 转换到 CIE LAB 色彩空间（感知均匀）
4. 计算该 LAB 值与色卡中每个颜色的 Delta E 距离
5. 取距离最小的颜色作为匹配结果

### 白色检测

```
if (L* > threshold && |a*| < 8 && |b*| < 8) → 标记为空白
```

- `threshold` 由灵敏度滑块控制（70~98，默认 90）
- a*、b* 接近 0 表示颜色接近中性灰/白

### 钉板粗线

每 `boardSize`（默认 28）格绘制一条红色粗线（`medium` 样式），与物理钉板对齐，方便拼豆时定位。

## 添加新色卡

1. 在 `palettes/` 目录下创建新文件，如 `perler.js`
2. 按照现有格式定义色卡数组：
   ```javascript
   const PALETTE_PERLER = [
     { code: 'P01', rgb: [255, 255, 255] },
     // ...
   ];
   ```
3. 在 `index.html` 中引入脚本：
   ```html
   <script src="palettes/perler.js"></script>
   ```
4. 在 `app.js` 中注册到 `PALETTES` 对象：
   ```javascript
   const PALETTES = {
     'artkal-s': PALETTE_ARTKAL_S,
     'mard':     PALETTE_MARD,
     'perler':   PALETTE_PERLER,
   };
   ```
5. 在 HTML 下拉菜单中添加选项：
   ```html
   <option value="perler">Perler (XXX色)</option>
   ```
